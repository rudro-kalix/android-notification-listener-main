Android • Kotlin • Firestore

Automatic Payment Gateway

Capture transaction notifications, parse the essentials, and sync them to Firestore so your web dashboards and backend workflows stay in lockstep.

View on GitHub Designed for educational and experimental use.

Core Highlights

  • Listens to incoming transaction notifications.
  • Extracts transaction ID, amount, sender, and package name.
  • Syncs data to Firestore for easy integration.

Use Cases

Automatic Verification

Keep payment statuses updated automatically for websites and dashboards without manual checks.

Realtime Reconciliation

Match incoming payments with backend records for faster settlement.

Mobile-to-Web Bridge

Connect Android payment notifications to web apps and services with a lightweight pipeline.

Notes & Limitations

  • Requires Notification Access permission on-device.
  • Works only with payment apps that show transaction details.
  • Built for personal, educational, and experimental use.

This project respects security boundaries and only processes user-visible notification content with explicit permission.
